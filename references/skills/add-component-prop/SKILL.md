# Add Component Prop

## When to Use

When adding a new prop to an existing React component — reads the component,
applies the change, updates the corresponding test file, and runs targeted tests.

This skill was auto-generated by the skill-extractor from a repeated session pattern.

## Workflow

1. **Read the component** — Use `read` to examine the target component file and understand its current props interface.
2. **Edit the component** — Use `edit` to add the new prop to the interface/type definition and implement it in the JSX.
3. **Read the test file** — Use `read` to examine the corresponding `*.test.tsx` file and understand existing test patterns.
4. **Edit the test file** — Use `edit` to add test cases covering the new prop (default value, custom value, edge cases).
5. **Run targeted tests** — Use `shell` to run `npm test -- --filter=<ComponentName>` and verify all tests pass.

## File Patterns

- `src/components/*.tsx` — React component files
- `src/components/*.test.tsx` — Corresponding test files

## Rules

- Always update both the component and its test file in the same workflow
- Follow existing prop naming conventions (check other props in the interface)
- Add at least one test for the default prop value and one for a custom value
- Run only targeted tests, not the full suite, for faster feedback
